# Universe Explorer — 全案設計框架 & 目標藍圖

> 從 v0(已建成)拉高到整個案子的視角。
> 這份文件回答三件事:**要往哪去(目標階層)**、**用什麼骨架去(設計框架)**、**分幾步到(演進路線)**。
> 上位憲法以 v0 Kickoff Spec 為準,本文件不覆蓋、只延伸。

---

## 0. 北極星(North Star)

> **建立一個「誠實區分人類已知與未知」的科學知識探索系統 —— 它不給答案,它給出知識的形狀。**

判斷任何功能該不該做,只問一句:**它讓「已知/未知」的邊界更誠實、更可回溯了嗎?** 若否,不做。

---

## 1. 目標階層(Goal Hierarchy)

由上而下,越上位越穩定,越下位越可調整。

| 層級 | 目標 | 衡量 |
|---|---|---|
| **使命 Mission** | 讓一般人能誠實看見「科學到底知道什麼、還不知道什麼」 | 質性:讀者能一眼分辨地基與天花板 |
| **產品目標 Product** | 把「證據 → 燈號 → 呈現」這條認識論管線,做成可跨領域複用的引擎 | 換一個 Data 層就能長出新領域,核心不動 |
| **系統目標 System** | 四層依賴不可逆、每個宣稱可回溯到真實出處、機械把關憲法 | validator 零違憲、隨機抽 claim 可被第三者覆核 |
| **階段目標 Phase** | 見 §6 Roadmap,每版證明一件事 | 每階段有獨立驗收條件 |

### 反目標(Non-Goals — 誠實劃線,防止範圍蔓延)

這些**不是待辦、是刻意不做**,寫下來是為了守住焦點:

- ❌ 不做「AI 宣布事實」的問答機器人 —— 那是這個系統的**反面**。
- ❌ 不做擬人化科學家辯論配音 —— Evidence Debate 永遠用**結構化欄位**,不用 persona。
- ❌ 不追求資料完整性優先於認識論正確性 —— 寧可少收錄,不可假精度。
- ❌ 不做娛樂性 3D 視覺當賣點 —— 視覺服務於「看見知識形狀」,不是特效。
- ❌ 不讓 AI 自我認證(self-certification)—— 判定必須攤在證據上、可被推翻。

## 2. 核心設計框架(五支柱)

整個系統站在五根柱子上。任何新功能都應該能被歸到其中一柱,並且不得違反其他柱。

```
┌─────────────────────────────────────────────────────────────┐
│  ①認識論 Epistemics   ——  系統的「憲法」,決定什麼算「知道」      │
│  ②架構 Architecture   ——  四層單向依賴,AI 墊底                 │
│  ③領域 Domain Model   ——  Topic→Claim,與題材無關的地基         │
│  ④產品 Product/UX     ——  把知識的形狀「看得見」                 │
│  ⑤治理 Governance     ——  誰判燈、如何被推翻(validator 當法院)  │
└─────────────────────────────────────────────────────────────┘
```

### ① 認識論支柱(系統的核心 IP)

- **Reference First, AI Last** —— 出處先於一切,AI 最後。
- **燈號屬於 Claim,不屬於 Topic** —— 樞紐設計,不可退回。
- **五格光譜 + 機器可查的入格條件** —— 判定不是形容詞,是可勾稽的條件清單。
- **禁止假精度** —— 沒有 73% 這種捏造連續數字;確定性從證據內容湧現。
- **雙軸張力(已知未解)** —— 「共識強度」與「證據強度」目前壓在同一條光譜,會分岔(如 Hawking radiation)。v0 記錄不硬解,**雙軸拆分是本框架已標記的未來主線**(見 §5)。

### ② 架構支柱(四層,單向不可逆)

```
Data Layer ──▶ Evidence Layer ──▶ Knowledge Layer ──▶ AI Narrative Layer
 (出處)          (掛出處的證據)       (Claim+燈號)         (只整理,不宣布)
```

AI 在**最底層**。拿不到 Evidence 層的真實出處,就不准生成上層。這條方向是系統的物理定律。

### ③ 領域模型支柱(可複用的地基)

`Topic(容器)→ Claim(最小單位,掛燈號)` 這組結構**與題材無關**。
換掉 Data 層的來源(arXiv→海洋觀測資料庫),同一套引擎就能長出 Ocean Explorer。
**這是本案能從「黑洞 demo」變成「平台」的關鍵。**

### ④ 產品/UX 支柱

- 核心體驗 = **看見知識的形狀**(哪塊硬、哪塊在摸索),不是讀一篇答案。
- 燈號地基在上、天花板在下;證據可展開、open questions 讓讀者自己數。
- 結構化的 Evidence Debate 取代擬人辯論。

### ⑤ 治理支柱(誰判燈)

| 版本 | 判定方式 | 防濫用機制 |
|---|---|---|
| v0 | 人工判定 | status_reason 強制逐條對應入格條件,可被第三者推翻 |
| v1+ | 半自動提案、人工確認 | AI 只能「提議燈號 + 附證據」,人拍板 |
| 遠期 | 條件觸發的自動判定 | 每次自動判定都留稽核軌跡,可回溯、可覆核 |

**validator 是這個系統的憲法法院** —— 它不判對錯,它擋違憲。

## 3. 四層架構 × 版本演進

每一層在不同版本的「成熟度」不同。這張表是工程排序的依據。

| 層 | v0(已建) | v1 | v2+ |
|---|---|---|---|
| **Data** | 人工填入真實出處 | 自動抓取(arXiv / NASA / ESA),含快取與版本 | 多來源、跨語言、可信度分級 |
| **Evidence** | 手工掛 source_ref | 抓取結果自動結構化為 evidence,強制掛來源 | 證據去重、衝突偵測 |
| **Knowledge** | 人工判燈 + 可回溯 | 半自動燈號提案(人確認)+ **雙軸拆分** | 條件觸發的燈號遷移(status_history 自動長出) |
| **AI Narrative** | **不生成**(只攤結構) | 受約束生成:每句可回溯到 evidence | 對比敘事、時間軸敘事 |

> 注意 v0 的 AI Narrative 層是空的 —— 這是刻意的:先把下三層的地基證明穩,才往上蓋敘事。

## 4. 燈號系統的演化(單軸 → 雙軸 → 半自動 → 自動)

這是整個案子在「認識論」上最重要的一條演化主線。

```
v0  單軸五格 + 人工判定
      │  已標記張力:共識強度 vs 證據強度會分岔
      ▼
v1  雙軸拆分(consensus_strength × evidence_strength)
      │  Hawking radiation 這類 claim 不再被硬塞成一個燈
      ▼
v1  半自動判定:AI 依入格條件「提案」燈號 + 附證據,人確認
      │
      ▼
v2+ 條件觸發的自動遷移:新證據滿足遷移條件 → 自動更新燈號 + 寫 status_history → 觸發推送
```

**設計原則**:自動化程度可以升,**可回溯性不准降**。任何一版,拿掉 status_reason 都要能覆核。

## 5. Roadmap(分階段:每階段證明什麼、解鎖什麼、驗收)

| 階段 | 一句話 | 證明 / 解鎖 | 驗收線 |
|---|---|---|---|
| **P0 ✅ 2026-07-09** | 用黑洞跑通整套認識論 | 證明:認識論在真實題目上站得住 | 憲法零違反、4 claim 出現 ≥3 燈號、可回溯、知識形狀可見 |
| **P1 ✅ 2026-07-10** | arXiv 真實抓取(cite ⇒ fetch) | 解鎖:Data 層自動、可回溯地長出來 | 8 個引用全量可回溯到 endpoint、竄改快取抓得到、候選閘成立(NASA/ESA 留 P1+) |
| **P1.5 ✅ 2026-07-10** | 雙軸拆分:共識(人工)× 證據(機械湧現) | 解鎖:Hawking 型分岔成為結構事實 | 證據軸無宣告欄位、推導可重算、分岔頁面可見 |
| **P2 ✅ 2026-07-10** | 機械提案引擎(A 路線)、人拍板 | 解鎖:提案→人審→稽核的治理管線(LLM 提案器可日後替換) | 提案不落地、human 條件絕不猜、機械只排除不核准、append-only 稽核 |
| **P3 ✅ 2026-07-10** | 快照 diff → 事件;無聲改燈違憲 | 解鎖:從「快照」變「活的」知識(推送管道留 P5) | 穩定零噪音、事件回指推導、未記錄遷移擋 build |
| **P4 ✅ 2026-07-10** | 換 Data 層長出海洋題材 | 證明:領域模型真的與題材無關 | 六個引擎檔雜湊零改動、海洋 4 claim 出處全網路核實、兩 topic 並存、所有把關一體適用 |
| **P4b ✅ 2026-07-10** | 第三領域(系外行星),arXiv 來源再壓測 P1 | 證明:新領域自動繼承全部把關(test_registry 對未來領域永久生效) | 6 筆新 arXiv 來源核實+抓取、Planet Nine 第二個真實 Competing、8 筆 claim_added 事件實證 P3 |
| **R6 ✅ 2026-07-10** | AI Narrative 層回填(第四層) | 解鎖:組稿器/檢查器分離 —— 未來 LLM 組稿走同一個法院 | 逐句 refs 強制、憲法開場句式、% 僅限逐字引用證據、把關失敗整段寧缺勿越權 |
| **P5 ◐ 進行中** | events → outbox digest(推送管道) | 已解鎖:變化可送達訂閱者(檔案即介面) | digest 只轉述不詮釋、逐行回指事件檔、無事件即靜默。**未做**:SMTP/webhook 傳輸、NASA/ESA adapter、效能 |

**排序理由**:先把「不准編、可回溯」的地基(P0)證明穩,再上「自動抓取」(P1,最重),再上「更聰明的判定」(P1.5/P2),最後才「活起來 + 跨領域」(P3/P4)。**能力可以往後加,誠實不准往前退。**

## 7. 成功的定義(不同層,不同標準)

刻意分開,避免「v0 通過」被誤讀成「系統可用」。

- **認識論成功** = 隨機抽任一 claim,第三者能拿入格條件覆核並可能推翻。(P0 已達)
- **工程成功** = 資料能自動、可回溯地流過四層,validator 全程零違憲。(P1 目標)
- **產品成功** = 使用者能一眼說出某主題「哪裡是地基、哪裡是天花板」。(持續)
- **平台成功** = 換一個領域,核心引擎零改動。(P4 目標)

> **一句總結**:這個案子賣的不是「答案」,是「一套可被推翻的、誠實的知識結構」。
> 所有工程都在服務這一件事;任何讓系統看起來更聰明、卻更不可回溯的功能,都是走反方向。
