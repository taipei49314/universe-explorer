# 憲法修正案 #2 — 敘事層在地化(Localization 接縫)

> 定位:第二次引擎修改(narrative.py)。中文版 artifact 目前省略 AI 敘事區塊,
> 因為組稿器只會英文,而在呈現層自行翻譯組好的句子會**繞過法院**(check),違憲。
> 正解:在引擎開一個 Localization 接縫,讓任何語言的敘事都走同一個 check()。

---

## 0. 立法原意

R6 的核心從來不是「敘事是英文的」,而是:
**逐句掛 refs、憲法開場句式、% 僅限逐字引用證據、組稿器與檢查器分離。**
語言是呈現屬性;把關是結構屬性。修正案讓語言可換,把關不換 ——
與 P2(提案器可換)、R6(組稿器可換)同一個設計哲學,這是第四個「聰明可換、法院不換」接縫。

## 1. 修正條文

1. `narrative.py` 增加 `Localization` 協定(開場句式 + 各句型模板 + 證據文字取用),
   預設 `EnglishLocalization` 行為與現行**逐字元一致**(不帶 loc 呼叫 = 現狀)。
2. `compose(claim, loc)` 與 `check(claim, sentences, loc)` 同時吃 loc:
   - 開場句式檢查改用 `loc.opening`(中文憲法句式:「根據目前收錄的證據」)。
   - % 逐字引用規則改對 `loc.evidence_text(...)` 檢查 —— 中文敘事引用中文證據描述,
     一樣必須逐字。
3. 中文 Localization 住在 **data 層**(translations_zh.py),不住引擎 ——
   引擎只認識協定,不認識任何語言。
4. **翻譯本身的誠實邊界**:識別碼(refs、claim id、E1-E5 代號)不翻;
   缺翻譯退回英文,可見地降級,絕不捏造。

## 2. 驗收

1. 不帶 loc 的既有呼叫產出與修正前逐字元相同(回歸零破壞)。
2. 中文敘事通過同一個 check():逐句 refs、中文開場句式、% 逐字引用(測試)。
3. 偽造的中文壞敘事(無 refs / 錯開場 / 非逐字 %)被同一個法院抓(測試)。
4. 中文版 artifact 補上 AI 敘事區塊,重新發佈同一 URL。
5. 全部測試套件照過;引擎基準重新蓋章。
