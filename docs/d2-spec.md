# D2 Spec — 探索 UX(從「攤開的資料」到「可探索的知識」)

> 原則不變:前端陽春優先、結構清楚;視覺服務於「看見知識形狀」,不是特效。
> 全部靜態、零外部依賴(無 CDN、無框架)—— 跟資料一樣自足。

## 1. 範圍

1. **單 claim 永久連結**:每個 claim 卡片有穩定錨點(`#c-<claim_id>`),
   卡片上有可點的 ¶ 連結。任何人可以直接分享「就這一個宣稱」。
2. **探索總覽頁**(`explore.html`):三領域 12 個 claim 壓成緊湊卡片
   (燈號、標題、領域、證據軸、分岔章),支援:
   - 依**燈號**篩選(五格 chips,可多選)
   - 依**證據軸**篩選(E1-E5 chips)
   - **文字搜尋**(標題 / id / 領域,即打即濾)
   - 卡片點下去跳到該 claim 的領域頁錨點
3. **開放資料**(`claims.json`):全部 claim 的機器可讀匯出 —— 燈號、雙軸、
   status_reason、evidence(含出處與分級)、open_questions、sources。
   **只匯出已收錄欄位,不新增任何推導以外的數字。** 這是未來第三方覆核的介面。

## 2. 誠實邊界

- 篩選計數(「顯示 N / 12」)是可見清單的計數,合憲(讀者可自行重數)。
- explore 頁先做英文;中文版沿用單頁 zh.html,登記為後續。
- 不做排序演算法、不做推薦 —— 排序永遠是固定的:地基在前、天花板在後。

## 3. 驗收

1. 每個 claim 錨點穩定存在;permalink 可直達(測試:HTML 含 `id="c-<id>"`)。
2. explore.html 含全部 12 claim + 篩選 chips + 搜尋框;純 vanilla JS、零外部資源。
3. claims.json 可解析、12 筆、欄位完整、燈號與引擎一致(測試)。
4. 全部套件照過;CI 部署後線上可用。
