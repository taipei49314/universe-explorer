# T4 Spec — LLM claim 起草管線(B3,接縫的集大成)

> 內容規模化的引擎。本地 LLM 起草**新 claim 的骨架**,但每一步都踩在
> 既有法院上:v0 憲法、受控詞彙、cite⇒fetch、證據軸推導、提案相容集、
> 敘事檢查 —— **草稿在人看到之前,先過完全部機械法**。人審的只剩一件事:
> 內容是否忠於來源(真實性),因為合法性已被機器擔保。

## 0. 治理鐵律(沿用,零新法院)

1. **來源由人選。** CLI 輸入 = 人挑好的 arXiv id(從 candidates 或自查);
   LLM 拿不到「選來源」的權力。
2. **Prompt 只餵逐字抓取的官方 metadata**(標題/摘要),抓取走既有
   cite⇒fetch 管線 —— LLM 讀到的每個字都有 provenance。
3. **燈號建議必須落在機械相容集內。** 起草的證據先機械推導證據軸、算 P2
   相容集;LLM 建議的燈號不在集內 → 整份草稿拒收。
   machine 條件的理由由推導自動填,**不經 LLM**;human 條件照 B2 規矩:
   LLM 草稿 + UNVERIFIED 章。
4. **草稿永不落地。** 通過全部法院的草稿寫入 `drafts/`(物理隔離),
   附完整法院報告;沒通過 → 什麼都不寫(寧缺勿違憲)。
   人審核真實性後,自己把 Python 寫進 `data/` —— 屆時全部把關再跑一次。

## 1. 範圍

- v1 只收 arXiv 來源起草(官方 API 有摘要全文;Crossref metadata 常無摘要,
  DOI 來源起草留 T4+)。
- 失敗即退回:模型沒開/格式壞/任一法院不過 → 印違憲清單,零產出。

## 2. 驗收

1. **合法性先於人審**:活體驗證一筆真實草稿,產出的 drafts/*.json 附
   「validator 零違憲 + 證據軸推導 + 相容集 + 敘事檢查通過」的法院報告。
2. **法院咬 LLM 照舊**(離線測試):裸 % 草稿殺、燈號出相容集殺、
   詞彙外 evidence type 殺、引用未抓取來源殺。
3. **不落地**(測試):管線跑完,data/ 位元不變;drafts/ 每筆 UNVERIFIED。
4. 全部套件照過。
