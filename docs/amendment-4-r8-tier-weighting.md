# 憲法修正案 #4 — 來源分級進入證據軸(銷 R8)

> 修正案 #3 建了分級但刻意不動證據軸。本修正案完成它:
> **「獨立驗證」是對「同儕審查記錄」的宣稱** —— 未經審查的來源不能建立獨立性。

## 0. 修正條文(E1 規則收緊,其餘不動)

| 等級 | 修正前 | 修正後 |
|---|---|---|
| **E1 多重獨立直接** | ≥2 條直接觀測、掛不同出處 | ≥2 條直接觀測、掛 **≥2 個不同的 PRIMARY 出處** |
| E2 單線直接 | 不變 | 不變(非 PRIMARY 的直接觀測仍可得 E2 —— 它仍是直接觀測,只是單線,誠實) |
| E3/E4/E5 | 不變 | 不變 |

推導理由(`Derivation.reasoning`)必須寫出 PRIMARY 直接出處的計數 —— 可回溯照舊。

## 1. 對現有資料的影響:零

現有三領域所有 E1 claim(event_horizon、vents、exoplanets_exist)的直接觀測
全數掛 PRIMARY 出處 —— 修正案改的是**法律**,不是任何現有判定。
規則咬人與否由偽造測試證明(兩條 PREPRINT 直接觀測 → E2 而非 E1)。

## 2. 工程註記

tier 表從 validator.py 移居 model.py(分級屬於 taxonomy,住 schema 層);
validator 保留 re-export,呼叫端不變。本修正案觸及 model / validator / axes 三檔,
全部重新蓋章。

## 3. 驗收

1. 現有 12 claim 證據軸全數不變(回歸)。
2. 偽造:2 條 PREPRINT 直接觀測 → E2;1 PRIMARY + 1 PREPRINT → E2;2 PRIMARY → E1(測試)。
3. 推導理由含 PRIMARY 計數,第三者可重算(測試)。
4. 全部套件照過。

# 憲法修正案 #5 — 違憲報告標註法源(銷 R5)

> 規模化後,「你違憲了」不夠 —— 要說「你違了哪條法、哪次修正案定的」。

## 條文

每條違憲規則登記其法源(`LAWS`:rule → 出處文件),違憲訊息自動附註,如:
`[firewall] no_fake_precision (law: v0-constitution §3): ...`
未登記法源的規則 = 測試失敗(法不可無源)。

## 驗收

1. 每條現行規則(v0 + 修正案 + P1 + P3)都有法源(測試:LAWS 覆蓋全部規則名)。
2. 違憲輸出含法源字串(測試)。
