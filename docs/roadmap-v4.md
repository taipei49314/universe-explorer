# Roadmap v4 — 成熟與開放(2026-07-10 規劃)

> v3 收斂時的系統:活的(週巡邏、feed)、可挑戰的(issue 管道)、可規模化的
> (T4 管線 + T5 節奏)。v4 的主軸:**把「一個人的工程」變成「經得起世界參與的
> 公共物」** —— 法律地位、規範彙編、以及讓活性覆蓋到最後的盲區。

## 盤點發現的真實缺口

- **G4:repo 無 LICENSE。** 公開邀請挑戰與覆核,卻未授權任何人使用 ——
  預設「保留所有權利」,連 fork 覆核都在灰區。這與專案精神直接矛盾。
- G5:規範散落在 v0 spec + 六個修正案 + 各階段 spec 裡;LAWS 登記表是機器讀的,
  人要湊全貌得讀十份文件。缺一份「憲法彙編」。
- G6:arXiv 快取的論文出新版本(v2/v3)時系統無感 —— T1 只看 Crossref 的
  正式更正;arXiv 修訂是另一種「來源在動」。

## v4 項目(依序)

### V4-1 授權條款(需 Nelson 拍板 —— 法律決定,不代辦)

建議:**程式碼 MIT;內容與資料(claims/敘事/翻譯)CC BY 4.0**。
理由:MIT 讓引擎最大化被複用(認識論引擎本來就想跨領域繁殖);
CC BY 讓 claims 資料可被引用、轉載、覆核,只要求署名 —— 與「可挑戰」精神一致。
替代:全 MIT(最簡)/ 內容 CC BY-SA(要求衍生同授權,更「傳染」)。

### V4-2 憲法彙編(constitution.md)

- 單一權威文件:四層架構、五格條件、雙軸、全部 23+ 條規則(附法源與白話說明)、
  修憲程序(條文→修法→新舊測試→重新蓋章→commit 註明編號)正式成文。
- 一致性測試:彙編中列的規則集 == LAWS 登記表(文件與程式碼不准漂移)。
- 網站 about 頁連過去;成為挑戰者的判例法總覽。

### V4-3 arXiv 版本監測(T1b)

- 週巡邏擴充:對每個 arXiv 快取比對 API 的最新版本號;出新版 → 事件 → feed →
  issue(與 Crossref 巡邏同一 workflow)。機械記錄:id、舊版、新版、日期,零詮釋。
- 誠實邊界:偵測「版本變了」,不判斷「結論變了沒」—— 那是人的重審。

### V4-4 T4b 分步起草(讓 7B 做多來源)

- 把多來源起草拆成「每來源一次呼叫產一條證據 → 機械合併 → 條件補完 → 全院」。
  免費工程修復;14B 模型仍列為可選加速(需 Nelson 拉模型)。

### V4-5 T5 續跑(常態)

- 節奏照舊;middle-term 目標:30+ claims 或第四領域(候選:地震學 —— USGS 開放
  資料、真實 Competing 案例多)。
