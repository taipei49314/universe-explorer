# 憲法修正案 #3 — 來源可信度分級(C1,銷 R3 第一步)

> 動機:R3 登記的張力 —— 自動抓取時代,來源可信度不一。目前 Source.kind 是自由文字,
> 「同儕審查論文」與「預印本」在系統眼裡等值。這在人工精選時代無害,規模化後是漏洞。

## 0. 立法原則

1. **分級是機械分類,不是品質評分。** 每個來源依 kind 文字歸入恰一個等級;
   等級是離散類別,禁止數字分數(那是假精度的另一張臉)。
2. **可分類是憲法要求**:kind 無法歸類 → 違憲(`unclassifiable_source_kind`)。
   規模化後任何新來源都被迫誠實申報自己是什麼。
3. **本修正案只做分類與可見性,不動證據軸權重。** 讓分級影響 E1-E5 推導
   (如「預印本不得計入獨立驗證」)是實質認識論變更,登記為 R8,需獨立修正案。

## 1. 分級表(關鍵詞先到先得,詞彙集互斥)

| 等級 | kind 含關鍵詞 | 意義 |
|---|---|---|
| `PRIMARY` | "peer-reviewed" | 一次文獻(含 collaboration result、critique) |
| `SECONDARY` | "textbook" / "prize citation" | 權威二次記錄 |
| `PREPRINT` | "preprint" | 未經審查的預印本 |
| `DATASET` | "dataset" / "archive" | 資料記錄 |

## 2. 驗收

1. 現有 12 claim 的全部來源可分類且無歧義(測試)。
2. 塞一個 kind="部落格文章" 的來源 → `unclassifiable_source_kind`(測試)。
3. 頁面上每個來源顯示等級章;分級不出現任何數字分數。
4. 全部套件照過;引擎基準重新蓋章(validator.py)。
5. R8(分級 × 證據軸權重)登記入張力表。
