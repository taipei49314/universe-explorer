# P1.5 Spec — 雙軸燈號拆分(共識強度 × 證據強度)

> 定位:Roadmap P1.5。解掉 v0 登記在案的張力 R1:
> 「共識強度」與「證據強度」壓在同一條光譜,遇到 hawking_radiation 型分岔只能在
> status_reason 手動註記。P1.5 把它拆成兩條軸,分岔從「文字註記」升級為「結構事實」。

---

## 0. 樞紐設計決定(本版的靈魂)

兩條軸的**判定機制刻意不同**:

| 軸 | 是什麼 | 誰判定 | 可回溯機制 |
|---|---|---|---|
| **共識軸** | 五格燈號(沿用 v0,不動) | 人工判定 | status_reason 逐條對應入格條件 |
| **證據軸** | 已收錄證據的強度等級 | **沒有人判定 —— 機械推導** | 推導規則公開、可重算;改證據 → 軸自動變 |

證據軸**不存在宣告欄位**。它是 `derive()` 函式對 evidence 記錄(每條都掛真實出處)
跑一組公開規則的輸出。這是憲法第 3 條「確定性必須從證據欄位的內容湧現,不得由 AI 宣告」
的字面實作:想改變證據軸,唯一的路是收錄新證據(而新證據又被 P1 的 cite=>fetch 擋著)。

## 1. 證據軸分級(離散五格,禁止連續數字)

由上而下,滿足最強的一格即歸該格。全部從 evidence 記錄的 `type` 與 `source_ref` 機械判定:

| 等級 | 條件(機械) |
|---|---|
| **E1 多重獨立直接** | ≥2 條 `direct observation`,且掛**不同**出處 |
| **E2 單線直接** | 恰 1 條 `direct observation`(或多條但同出處) |
| **E3 間接/類比** | 無直接觀測,但有 `indirect observation` 或 `analog experiment` |
| **E4 純理論** | 只有 `theoretical derivation` / `theoretical result` |
| **E5 無** | 無證據(本來就違憲,不會出現在合法資料中) |

`Evidence.type` 從自由文字升級為**受控詞彙**(上述五種),validator 新增檢查:
詞彙外的 type → 違憲(`invalid_evidence_type`)。沒有受控詞彙,機械推導就是沙上建塔。

## 2. 分岔(divergence)的機械定義

共識軸在 {Established, Strong} **且** 證據軸在 {E3, E4, E5} → 標記「分岔」。
這不是誰的意見,是兩個可回溯事實的並置。頁面上必須一眼可見。

## 3. 範圍劃線

### 做
1. `axes.py`:證據軸 enum + 受控詞彙 + `derive()`(輸出等級 **和** 逐條推導理由,可回溯)。
2. validator:受控詞彙檢查。
3. 頁面:每個 claim 顯示雙軸;分岔 claim 有明確標記;推導理由可展開。
4. hawking_radiation 的 status_reason 裡那段手動 SPLIT 註記改寫 —— 分岔已是結構事實,
   註記只需服務入格條件本身。

### 不做
- ❌ 共識軸的判定機制不動(仍人工,P2 才半自動)。
- ❌ 不給兩軸算加權總分(那是假精度的高級版)。
- ❌ 不動 Claim dataclass:證據軸是**推導值**,不是儲存欄位 —— schema 零改動。

## 4. P1.5 驗收(先於實作)

1. **雙軸各自可回溯**:共識軸沿用 status_reason;證據軸的 derive() 附逐條推導理由,
   第三者可拿 §1 規則表重算並推翻。
2. **分岔結構化可見**:hawking_radiation 在頁面一眼看出「共識 Strong × 證據 E3」+ 分岔標記,
   不再依賴 status_reason 的文字註記。
3. **湧現不宣告**:程式中不存在任何「填寫證據軸」的欄位或參數;測試證明改動證據記錄
   會自動改變推導結果。
4. **零違憲不退步**:v0 + P1 全部測試照過;受控詞彙檢查會咬人(測試證明)。
5. **同向 claim 不受干擾**:event_horizon(🟢×E1)、firewall(🔴×E4)照常呈現,無分岔標記。
